# Algorithmique et Programmation  

## 1. Introduction à l’algorithmique

### Qu'est-ce qu'un algorithme ?

Un **algorithme** est un procédé systématique, défini, et non ambigu qui permet de résoudre un problème ou d’accomplir une tâche en un nombre fini d’étapes.

Exemple :
> Une recette de cuisine est un algorithme : elle décrit, étape par étape, comment transformer des ingrédients bruts en un plat préparé.

## 2. Définitions fondamentales

| Terme | Définition |
|------|------------|
| **Instruction** | Unité de base d'un algorithme (ordre élémentaire). |
| **Programme** | Traduction d’un ou plusieurs algorithmes dans un langage de programmation. |
| **Variable** | Conteneur nommé pour stocker de l'information. |
| **Constante** | Valeur fixe utilisée dans un programme. |

## 3. Représentation d’un algorithme

- **Langage naturel structuré**
- **Pseudo-code**
- **Organigramme (diagramme de flux)**

**Exemple de pseudo-code d’un algorithme simple :**

```
Début
  Demander à l’utilisateur un nombre entier N
  Afficher N * N
Fin
```

## 4. Structures de base d’un algorithme

### a) Séquence
Suite d’instructions exécutées dans l’ordre.

### b) Sélection (condition)
Exécution conditionnelle des instructions.

Exemple :
```pseudo
Si age ≥ 18 alors
  Afficher "Majeur"
Sinon
  Afficher "Mineur"
Fin Si
```

### c) Itération (boucle)
Répétition d’un bloc d’instructions.

Exemple :
```pseudo
Pour i de 1 à 10
  Afficher i
Fin Pour
```

## 5. Spécification d’un algorithme

Un bon algorithme doit être :
- Finite (fini)
- Défini (non ambigu)
- Efficace (réalisable dans un temps admissible)

## 6. Débogage et validation

- **Débogage** : processus d’identification et de correction des erreurs dans un algorithme ou un programme.
- **Validation** : vérification que l’algorithme résout correctement le problème posé.
